About

Most of my working day is spent figuring out what needs to be built, how it should be built, who should build it, and when it needs to ship — and increasingly, I leverage AI to keep up with the rapid pace. Hands-on time is rare.

Boudreaux Labs is how I stay honest. I use it to keep up with AI tooling and to actually build the infrastructure patterns I ask my team to own — GitOps delivery, pipeline-driven IaC, zero-manual-steps deploys. If I'm going to ask someone to own it, I should understand what it looks like end to end.

Most recently that's meant Claude Code as a collaborative engineer — infrastructure decisions, debugging, security review, all as a conversation in the IDE. It changes the feedback loop in a way that's hard to overstate.

This site runs on the infrastructure it describes — EKS, ArgoCD, GitHub Actions. Created and destroyed in a single pipeline run. No manual steps, ever.

How it operates

Core Tenets

01

Phoenix Environment

Nothing is patched in place. The cluster is destroyed and rebuilt from scratch on every cycle — if it can't be recreated by the pipeline, it doesn't exist.

02

Near-Zero Cost at Rest

SPOT instances, a single NAT gateway, minimal replicas. When the cluster is down, cost is essentially zero. The whole point is to learn without a running bill.

03

Fully Within My Domain

Personal AWS account, personal domain, personal GitHub org. No employer infrastructure, no shared accounts, no permission dependencies on anyone else.

04

Pipeline Up, Pipeline Down

One pipeline trigger brings the entire compute and app stack live. Another tears it all down. No manual steps — no console, no kubectl, no config drift. Done almost daily.

Architecture

How it's layered

Each layer depends on the one below it. The bottom never changes; the top can be torn down and rebuilt in minutes.

Ephemeral — Apps

App Deployments you are here

Apps running on the platform, managed by ArgoCD. This site is the first. Each app lives in its own repo, owns its own Dockerfile and Kubernetes manifests, and is onboarded via a single entry in the GitOps config. SRE owns the platform; developers own their apps.

02
Ephemeral — Platform

Compute Platform

EKS cluster with ArgoCD, AWS Load Balancer Controller, External DNS, and the full add-on stack. Spins up and tears down via a single GitHub Actions pipeline. ArgoCD provides the self-service layer — once the cluster is up, it automatically syncs and deploys every onboarded app from Git.

01
Permanent — Foundation

Persistent Infrastructure

The things that never get torn down: AWS account, Route53 hosted zone, ACM certificate, GitHub OIDC provider, and the IAM roles that let GitHub Actions authenticate to AWS without static credentials. Created once and left alone. Everything above this layer depends on it.
